Multivariate Anomaly Detection for Memory Leaks

Time-series • JVM Metrics • LSTM • System Reliability

Context

Long-running JVM applications often degrade silently due to memory leaks. Traditional threshold-based monitoring fails to capture gradual or multivariate anomalies.

Problem Framing

The challenge was to detect abnormal memory behaviour early — before out-of-memory errors occurred — using only system metrics.

Approach

Metric Selection
Heap usage, GC activity, allocation rate, and thread count.
Data Simulation
Injected controlled leak patterns to model real failure modes.
Modeling
Trained LSTM networks to learn normal temporal behaviour.
Evaluation
Compared against statistical baselines and density-based methods.

Signals & Insights

Gradual deviation in memory utilisation captured by the model before failure.

Outcome

The model consistently detected abnormal behaviour earlier than static thresholds, providing meaningful lead time for intervention.

Reflection

This project reinforced the importance of understanding system behaviour before choosing algorithms — models work best when grounded in context.
